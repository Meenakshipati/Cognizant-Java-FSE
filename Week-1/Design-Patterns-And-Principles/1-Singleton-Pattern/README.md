# Singleton Pattern

## Objective
Ensure a logging utility class has only one instance throughout the application lifecycle.

## Steps
1. Made constructor private
2. Created private static instance  
3. Provided public static getInstance() method
4. Tested that only one object is created

## Output
Only one Logger instance is created and reused.

## Concepts Used
Singleton Design Pattern, Lazy Initialization
